## 📊 Flow 4 — Re-engagement (para los que no se inscribieron)

Después de que alguien pide un recurso pero NO se inscribe después de 3 días.

**Mensaje único (72 horas después del último recurso pedido):**
```
{{first_name}}, te escribo para algo pequeño.

¿El recurso te sirvió? ¿O no era lo que buscabas? 🤔

Respóndeme con:
1️⃣ Me sirvió mucho
2️⃣ Me sirvió pero quiero más
3️⃣ No era lo que esperaba

Con tu respuesta afino los próximos. Es todo lo que necesito.

— Arie
```

**Si responde `1`:** mensaje "me alegra" + CTA al programa
**Si responde `2`:** CTA directo al programa
**Si responde `3`:** "¿Qué buscabas exactamente?" (abierto, Arie lee personalmente)

## 🔗 Instrucciones de configuración

### En ManyChat:

1. **Growth Tools → Keywords**
   - Crear un Keyword para cada palabra clave de la tabla
   - Match type: "Starts with" o "Is exactly" (recomendado `Is exactly` para evitar falsos positivos)
   - Action: Send Flow → seleccionar el flow correspondiente

2. **Automations → Flows**
   - Crear un Flow por cada tema (10 landings + 3 PDFs + post-registro + re-engagement)
   - Usar los mensajes de arriba tal cual

3. **Delays**
   - El Mensaje 2 de cada Flow requiere un Delay step de 30 min o 2 horas según especificado
   - El Mensaje 3 del Flow 1 requiere Delay de 24 horas + condición "abrió Mensaje 2"

4. **Tags (recomendado)**
   - Tag `leadmagnet_cafe`, `leadmagnet_desayunos`, etc. por cada recurso consumido
   - Tag `inscrito_reboot30` cuando completen el formulario
   - Tag `re_engagement` cuando cumplen 72h sin inscribirse

### Integración con FluentCRM (WordPress)

Cuando alguien completa el formulario en reboot30.html, el endpoint `reboot-master-api` ya registra el lead.
Para sincronizar con ManyChat:
- Opción A (recomendada): En FluentCRM, cuando se crea un contacto con tag `reboot_30`, disparar webhook a ManyChat API
- Opción B: Zapier entre el endpoint y ManyChat
